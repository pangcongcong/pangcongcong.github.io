css基础知识点梳理

本文仅梳理css基础知识点，包含平时常用或遗漏的css属性，不定期更新。

盒模型（CSS Basic model）

1.盒模型概念

在CSS中，使用标准盒模型描述这些矩形盒子中的每一个。这个模型描述了元素所占空间的内容。每个盒子有四个边：外边距边（Margin Edge）, 边框边（Border Edge）, 内填充边（Padding Edge） 与 内容边（Content Edge）。

如图：

盒模型分为IE盒模型和W3C标准盒模型。

  1. W3C 标准盒模型：
    属性width,height只包含内容content，不包含border和padding，也就是width=content；
  2. IE 盒模型：
    属性width,height包含border和padding，也就是width = content+padding+border。

这两种类型的切换在IE8+浏览器可以使用box-sizing属性控制
content-box：默认值，标准盒模型；
border-box：IE盒模型；

注：若不声明DOCTYPE类型，ie浏览器会解释为IE盒子模型，其他如FF会解释为W3C盒模型；声明DOCTYPE类型，则所有浏览器都会解释为W3C盒模型。

mpvue实战

项目背景：集樱花瓜分百万现金活动，需求主站内（h5）&小程序都可以玩，基于小程序传播&召回～所以需要h5与小程序两套代码实现，几乎全部功能都是重复实现。
项目git地址：git git@git.inagora.org:pangjingcong/sakuraMpvue.git

why mpvue

mpvue 优点概览

  • H5 代码转换编译成小程序目标代码的能力
  • 完整的 Vue.js 开发体验
  • 彻底的组件化开发能力：提高代码复用性
  • 方便的 Vuex 数据管理方案，方便构建复杂应用
  • 快捷的 webpack 构建机制：自定义构建策略、开发阶段 hotReload
  • 支持使用 npm 外部依赖
  • 使用 Vue.js 命令行工具 vue-cli 快速初始化项目

踩坑

1.不建议使用小程序的生命周期钩子
2.小程序页面传参
小程序传参：page onLoad 获取options
mpvue：this.$root.$mp.query在onLoad之后使用
3.不支持部分复杂的js表达式，三元表达式可以 //join() split()
4.不支持模板语法里面调用methods方法 //可以用computed
4.组件不支持Class与style绑定
5.onShareAppMessage同步问题 //先扣花瓣 再调分享
6.调试问题，报错未定位 //标签未闭合
7.推荐单独以mpvue的方式构建新的小程序，而非在现有小程序基础上新增
